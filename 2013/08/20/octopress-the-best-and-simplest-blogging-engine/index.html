Octopress - The Best and Simplest Blogging Engine

So here is my third attempt to setting up my blog. To be honest, its not really the setting up of blog, rather a promise to myself to keep myself regular at blogging. So the goal was to find a reasonably easier yet a very flexible blogging platform. I was just tired of wordpress and the likes. Being a ruby and nodejs enthusiast (apart from python), my search revolved around these languages, which would give me more power than a PHP based framework (PHP hasn’t caught my interest so far).

Searching on google gave a me a few decent options

Nodejs

Wheat – a blogging engine created by Tim Caswell, is one of the oldest Node.js blogging engines available. It takes a folder full of markdown formatted text files from git and renders them as HTML.
Scotch – is similar to Wheat in that it renders markdown pages. Unlike Wheat, however, it has a caching system built with the NoSQL datastore Redis.
Blacksmith – is another static site generator written in Node.js. Inspired by Jekyll, it aims to be a full content management system for those wanting a baked site.

Ruby

Typo – the oldest blogging platform in Rails. It has a full set of features you would expect from such an engine, which include powerful SEO capabilities, full themes, and plug-in extensions.
Jekyll – is a simple, blog aware, static site generator.
Octopress – is an obsessively designed framework for Jekyll blogging. It is easy to configure and easy to deploy.
Of the many, the above stood out as options. My criteria of evaluation was + Easy setup + Easy deployment + Easy to configure + Highly customizable

I picked up Octopress, for it fulfilled my criteria very well. Its built in Ruby and and is based on Jekyll (a static site generator). It builds on top of Jekyll by making it ready with template, theme and deployment options, which makes it a very easy to start with. Being built in ruby, its very easy to customize with minimal amount of code needed (thanks to a large array of functionality available on ruby).

The setup was as easy as cloning the octopress repository to my local drive with git and installing dependencies.


Once setup it just needs a bit of configuration for setting up your social media accounts, Google analytics account etc in a single file _config.yml. It is self explanatory.

For deployment, of the many choices, I chose to host it on my own server I took on linode. The configuration for deplyment is in Rakefile . For the deplyment option I chose rsync and filled appropriate field in the Rakefile.
